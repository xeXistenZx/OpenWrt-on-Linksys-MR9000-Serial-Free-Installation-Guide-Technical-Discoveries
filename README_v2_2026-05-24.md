### Correct Diagnosis (v2/v3)

**Both the EA8300 and MR9000 use the same chip — the Qualcomm QCA8075.**

The QCA8075 is not a switch — it is a **5-port Ethernet transceiver**. Using a 2-port transceiver like QCA8072 would not make sense for a 5-port device like the EA8300 or MR9000.

**The Real Reason EA8300 Firmware Fails on MR9000:**

The root cause is a **missing QCA8075 reset definition in the EA8300 Device Tree (DTS).**

1. The EA8300 DTS does not define a hardware reset for the QCA8075 transceiver
2. This works fine on EA8300 because its bootloader leaves QCA8075 in a compatible state
3. The MR9000 bootloader **pre-initialises the QCA8075 differently** — incompatible with OpenWrt
4. Without a reset defined in the DTS, OpenWrt cannot reinitialise the transceiver
5. Transceiver initialisation gets stuck → router halts
